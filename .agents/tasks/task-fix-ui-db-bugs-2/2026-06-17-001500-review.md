# Raw JSON purge and sleep tracker input validation (v2)

This change eliminates raw `JSON.stringify` dumps across 6 frontend pages, replaces them with formatted stat-card and key-value grid renderings, fixes blank result cards in NaturalLanguageSearch by using `resident_id` as the primary detection signal, corrects a literal HTML entity string for the clock emoji, and adds a 400-response guard to the sleep tracker controller when `residentId` or `sleepDate` is missing.

Watch for:
- **Object.entries iteration on unknown shapes** — every replacement iterates `Object.entries(data)` with a cosmetic filter (`!== 'id'`), which means if the backend returns deeply nested objects, arrays-of-arrays, or large blobs, the cards will render `[object Object]` or meaninglessly long strings. (likely)
- **No null/undefined guard before Object.entries** — several pages call `Object.entries(report as Record<string, any>)` without first checking that `report` is actually an object, relying on the surrounding conditional (`{report ? ...}`) which is truthy for empty arrays, `0`, etc. (possible)
- **SmartRota constraint field name guessing** — the card renders `c.contractHours || c.contract_hours` with no guarantee either field exists in the actual backend response shape; if neither is present, the entire hours row is invisible with no explanation. (likely)

## High-level view

The sleep tracker validation prevents a NOT NULL constraint violation by returning 400 early, but doesn't validate other fields (`qualityRating`, `totalSleepHrs`) that could similarly cause downstream 500s if the schema enforces NOT NULL on them.

The SmartRota staff constraints panel replaces flat error-message rendering with structured cards that handle both camelCase and snake_case field variants. The contract between the backend response shape and what the card expects is implicit — no TypeScript interface, no runtime shape validation.

NaturalLanguageSearch uses `resident_id` as the primary detection signal (correct — the backend always includes it), with a generic `Object.entries` fallback grid replacing the `<pre>` dump. The generic path degrades for nested structures: objects render via `Object.values(value).join(', ')`, silently dropping keys.

The 6 remaining pages follow an identical `Object.entries` stat-card template. The pattern is consistent but carries fragility: inconsistent value formatting between pages (some show array `.length`, others show `.join(', ')`), and nested objects still fall back to `JSON.stringify` on some paths.

## Stat-card pattern fragility across 6 pages

Every replacement follows the same template:

```tsx
Object.entries(data as Record<string, any>)
  .filter(([k]) => k !== 'id' && k !== 'care_home_id')
  .map(([key, value]) => (
    <div key={key}>
      <div>{/* formatted value */}</div>
      <div>{key.replace(/_/g, ' ')}</div>
    </div>
  ))
```

The value formatting branch varies by page: null becomes `'-'`, numbers display directly or via `.toFixed(1)`, arrays become `.length` or `.join(', ')` depending on which page you're on, objects become `Object.keys(value).length` or `JSON.stringify(value)`. The inconsistency means the same type of data looks different across pages. A `genres` array showing "3" on one page and "jazz, classical, ambient" on another creates user confusion.

FriendshipMapper's array branch has a secondary concern: for seating objects, it renders inner items' nested objects via `JSON.stringify(value)`, which means if a seating suggestion contains a `preferences` sub-object, the user sees raw JSON — exactly the bug this change is meant to fix, just pushed one level deeper.

## SmartRota constraint card data mapping

The card renders hours data from both camelCase (`contractHours`, `hoursThisWeek`, `remainingHours`) and snake_case (`contract_hours`) field names using `||` fallbacks. If both formats are present with different values (e.g., during a backend migration), the card silently picks the camelCase variant. If neither is present, the entire hours row disappears with no indication to the user that data is missing.
